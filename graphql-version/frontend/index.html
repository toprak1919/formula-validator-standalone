# Formula Frontend Backend Status

📱 Frontend Validator

Libraries Used:
  • math.js (v12.2.1) - Expression parsing & evaluation
  • expr-eval (v2.0.2) - Safe expression evaluation
  • Native JavaScript RegEx - Pattern matching
Validation Checks:
  • ✓ Missing operators between numbers/variables
  • ✓ Double/adjacent operators (++, --, **, //)
  • ✓ Leading operators (except minus)
  • ✓ Trailing operators
  • ✓ Empty parentheses detection
  • ✓ Parentheses balance checking
  • ✓ Variable syntax validation ($variable)
  • ✓ Constant syntax validation (#constant)
  • ✓ Undefined variable/constant detection
  • ✓ Function existence checking
  • ✓ Decimal notation validation
  • ✓ Real-time syntax highlighting

⚙️ Backend Validator (.NET)

Libraries Used:
  • System.Text.RegularExpressions - Pattern matching
  • HotChocolate (v15.1.8) - GraphQL server
  • Custom C# validation logic
Validation Checks:
  • ✓ Missing operators between numbers/variables
  • ✓ Double operators detection
  • ✓ Leading operators (except minus)
  • ✓ Trailing operators
  • ✓ Empty parentheses detection
  • ✓ Parentheses balance checking
  • ✓ Invalid variable syntax ($$ or $123)
  • ✓ Invalid constant syntax (## or #456)
  • ✓ Undefined variable/constant detection
  • ✓ Basic function validation
  • ⚠️ Limited decimal validation
  • ⚠️ No expression evaluation

⚠️ Key Differences:

  • Expression Evaluation: Frontend uses math.js for full evaluation, backend only validates syntax
  • Function Support: Frontend can evaluate math functions (sqrt, sin, cos), backend treats them as errors
  • Decimal Handling: Frontend has comprehensive decimal validation, backend has basic checks
  • Error Messages: Frontend provides more detailed error messages with suggestions
  • Performance: Backend validation is faster but less feature-rich
